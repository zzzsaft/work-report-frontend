# UI 规范

## 原则

- 手机端为年龄偏大的生产人员设计，一个页面只突出一个主要任务。
- 使用白色/浅灰底、深蓝主色、橙色提醒、绿色完成、红色异常；状态必须同时有文字。
- 正文不低于 16px，主要信息 20-24px，计时 31-43px；普通点击区不低于 56px，主操作不低于 64px。

## 手机端

- 当前工序信息使用单列分隔行，不嵌套卡片。
- 产品编号直接展示；长工序备注通过“查看工序备注”打开底部抽屉。
- 暂停使用确认抽屉；完工按照片、可选信息、确认提交组织。
- 暂停、待开始或无当前工序时可以切换工序；运行中不显示切换入口。
- 工序清单包含“历史 / 当前 / 未来 / 领取”。领取流程按“产品编号 → 部件编号 → 工序”逐级展开，领取、删除、去开始按钮保持大触控面积。
- 底部导航固定四项，并适配 iOS 安全区和 320px 宽屏幕。

## 管理端

- 深蓝侧栏搭配白色数据区。表格用于高密度信息，卡片只用于 KPI 或独立设置模块。
- 小组长导入页使用大文本框、预览表和错误行提示；高级分配页按产品、部件、工序选择，强制移除使用红色危险操作。
- 小于 760px 时侧栏变为抽屉；表格允许横向滚动。
- 图标统一使用 Lucide，统计图统一使用 Recharts。
